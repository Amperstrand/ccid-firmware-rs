# Smartcard and USB Pinout

Target MCU: STM32F469 (Discovery layout)

## Smartcard interface (ISO 7816)

| MCU Pin | Signal | Direction | Notes |
|---|---|---|---|
| `PA2` | `I/O` | Bidirectional | `USART2_TX` in smartcard mode, AF7, open-drain, pull-up |
| `PA4` | `CLK` | Output | `USART2_CK`, AF7, push-pull |
| `PG10` | `RST` | Output | Card reset control |
| `PC5` | `PWR` | Output | Card supply gate (`LOW = ON`) |
| `PC2` | `PRES` | Input | Card detect (`HIGH = card present`) |

## USB interface

| MCU Pin | Signal | Notes |
|---|---|---|
| `PA11` | `USB_DM` | OTG FS data- |
| `PA12` | `USB_DP` | OTG FS data+ |

## CCID identity

- USB VID:PID: `08E6:3437` (configured in `src/usb_identity.rs`)
- Advertised as `Gemalto IDBridge CT30`

## Electrical/behavior notes

- The smartcard I/O line is configured open-drain with pull-up.
- The clock is generated by USART smartcard mode (`CR2.CLKEN`).
- Card power/reset sequencing is implemented in `src/smartcard.rs`.
- Non-destructive hardware validation procedures are documented in `tests/hardware/README.md`.
